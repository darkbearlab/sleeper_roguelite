## 一句話定位
2D 俯視「**搜打撤**」(search → raid → extract) arcade。與 SLEEPER 同一套引擎，但**複製成獨立分支**維護——「很多行為和魅力應該從此分家」。

## 架構決定（使用者拍板）
- **複製分家**，不共用 `engine.js`、不破壞 SLEEPER 的單檔自足：動工時把當下的 `index.html` 複製到本目錄當起點，之後完全獨立改。
- 可重用基底（從 SLEEPER 帶來、再各自演化）：戰鬥/附身、敵 AI（IDLE↔ENGAGED FSM、巡邏 `patrolStep`、搜索/獵殺 `huntStep`＋共享情報、`bfsPath` 圖上尋路、近戰收尾衝刺）、撤離 `extractTick`、戰爭迷霧、打擊感 juice、地圖 registry＋編輯器（含 `editor.html`）、沙盒、I/O 接縫（Input/Sfx）、數值面板。

## 世界與勢力
- **玩家**：找寶、隨時可撤。
- **野怪**（＝改寫的警犬 `dog`）：巡邏。
- **住民**（＝巡邏的 `grunt`）：巡邏。
- **獵人**（各式人類敵人）：走訪 waypoint 網路、找到**有寶物標記的 waypoint**、拾取累值。
- **陣營（v1 決定）**：以**玩家本位**為基底（野怪/住民敵對玩家、以巡邏為主）；**獵人先做成 FFA**——彼此攻擊、見到其他勢力(玩家/野怪/住民/別的獵人)就打。完整敵對矩陣的細緻化暫緩，先 FFA。
  - 實作要點：把現有二元的「找最近敵人」目標選取（`nearestFriendly`/`nearestKnownPlayer`）一般化成「**最近的敵對他勢力單位**」；共享情報改為**逐勢力**。

## 核心機制
- **寶物**：waypoint 上的寶物標記 + 價值；玩家與獵人皆可拾取。傾向**每局隨機生成**（設計師標可生成點 → 每局抽子集 + 亂數價值）＝耐玩。
- **獵人尋寶 AI**：去最近**未被認領**的寶物點（沿用 `bfsPath` + 認領制）→ 拾取累積價值 → **`累積價值 ≥ 該獵人的滿意值` → 轉往撤離點離場**。每個獵人**滿意值不同**（有人小賺就走、有人貪心）。
- **玩家結算**：**無固定額度目標**；離場時統計「**消耗成本 vs 取得價值**」，盈餘＝得分。建議 `score = 撤出價值 − 耗用成本(彈藥/道具/…)`；**陣亡＝損失本局所得**（搜打撤精髓：死了什麼都沒了）。細節動工前定。
- **本機排行榜**：`localStorage`（線上排行榜之後再說，先 localhost）。
- （可選）**時間壓力**：待越久、獵人/危險越多（沿用 reinforce 計時）→ 強化「見好就收」的拉扯。

## 相對 SLEEPER 的新系統
1. **陣營/FFA 目標選取**（二元 → 多勢力；先 FFA）。
2. **寶物模型 + 拾取**（玩家 + AI）+ 編輯器「寶物」工具。
3. **獵人「尋寶 → 滿足 → 撤退」目標棧**（建在現有尋路/認領/撤離上）。
4. **成本/價值結算 + 本機排行榜**（新 meta + localStorage）。
5. **換皮**：野怪/住民/獵人（多為現有 dog/grunt + 新增人類敵人變體）。

## MVP（動工時的最小可玩核心）
一張圖：寶物每局隨機生成 → 玩家撿寶 + 撤離 → `score = 價值 − 彈藥成本` → 本機排行榜；野怪 + 住民巡邏並獵殺玩家；1–2 名獵人會尋寶 + 滿足撤退、且 FFA。跑通後再加陣營深度、撤離點伏擊、時間壓力。

## 暫緩 / 待議
線上排行榜；完整敵對矩陣（先 FFA）；成本與得分的精確公式；寶物生成的設計師可控程度。
