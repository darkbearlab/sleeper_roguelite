# SLEEPER — 開發進度與脈絡（handoff / compact 用）

> 本文件供對話 compact（壓縮）後接手用，盡量自包含。原始設計規格見 `SLEEPER_開發計畫.md`，實作全部在單一檔 `index.html`（原生 JS + Canvas 2D，零外部相依，瀏覽器開檔即玩）。所有可調數值集中在檔頂 `CONFIG` / `WEAPONS` / `ARCHETYPES` / `ABILITIES` / `ROADMAP`。

---

## 0. 一句話定位

俯視戰術射擊 × 附身式小隊操作 × Slay-the-Spire roguelite。一次只完整附身一名單位（FPS 式滑鼠鎖定瞄準、WASD 移動、Shift 奔跑），其餘隊員跑各自固定的「待機個性」本能；每場戰鬥在時間關門、追兵湧入前至少把一人帶到撤離點。撤出者帶入後續關卡、陣亡永久損失。在路線圖上選路前進（戰鬥/休息兩種節點），走到終點＝過關。

## 2. 場景狀態機（`scene`）

```
MAP(路線圖) ──選 COMBAT 節點──► COMBAT ──勝利(撤出≥1)──► 回 MAP（開放後繼節點）
     │                              └──全滅(撤出0)──► RUNEND
     └──選 REST 節點──► REST(二選一) ──► 回 MAP
走到無後繼的終點節點 ──► RUNWIN
```

- `scene ∈ { MAP, COMBAT, REST, ARMORY, RUNEND, RUNWIN, EDITOR }`。EDITOR＝關卡編輯器（路線圖右上鈕進入）；ARMORY＝軍械庫節點三選一。
- 流程函式：`initRun()`（開新 run，建隨機陣容→MAP）→ `selectNode(id)`（進節點）→ 戰鬥 `endMission()` / 休息 `restContinue()` → `advanceNode()`（標記完成、開放後繼或 RUNWIN）。
- **戰鬥在滑鼠鎖定後才開始推進**：`update()` 開頭 `if (scene!=='COMBAT') return; if (!pointerLocked) return;`（未鎖定＝暫停；Esc 解鎖＝暫停）。

## 4. 各系統詳解

### 4.1 附身 / 控制
- `controlledId`（players 索引或 null）。`possess(i)`：舊控制者→IDLE 並**記住 idleAnchor=facing**；新單位→CONTROLLED、**facing 與 aimDir 恢復到該單位 idleAnchor**（接手時朝它原本盯的方向，不是它飄到的角度）、清 followExtractId。切換無 CD（`switchHandoverMs=0`）。
- `toggleExtract()`（F）：被附身者→EXTRACTING、設 idleAnchor=facing、`controlledId=null`、`followExtractId=該單位`（鏡頭續跟）。
- `autoPossess(fromPos)`：**非撤離(IDLE)優先，無則退而附身撤離中(EXTRACTING)單位**（抓回手動）。
- **無空轉保險**（`update` step 7.5）：戰鬥中 `controlledId==null && followExtractId==null && 有 active 單位` → `autoPossess(null)`。刻意跟隨觀看期間（followExtractId 有值）不觸發。

### 4.2 旋轉鏡頭 + FPS 瞄準
- 鏡頭跟隨置中被附身單位、旋轉使其**朝螢幕上方**；`camTarget.angle = -π/2 - facing`，`cam` 以 followLerp/rotateLerp 平滑插值。
- **anchorY=0.66**：單位畫在畫面 66% 高（偏下）→ 前方視野更長。`applyCam`/`w2s` 用 `cv.height*anchorY` 當縱向錨。
- **瞄準＝Pointer Lock 滑鼠鎖定（mouselook）**：點擊鎖定，`mousemove.movementX × mouseSensitivity` 累積 `aimDir`，砲塔以**武器 turnSpeed** 追 aimDir（追不上＝轉向延遲；停滑鼠就停，無自轉）。
- ⚠️ 取捨：「恆朝上」與「即時絕對瞄準」數學上不可兼得，故為相對轉向，turnSpeed 是手感旋鈕。
- MAP/REST/結算用正常游標（不鎖定），`mouse.mx/my` 由 `canvasPos` 追蹤供點選。

### 4.3 戰爭迷霧（敵我共用）
- `canSee(observer, targetPos)` = 有視線(`clearLine`) ∧（在 `senseRadius` 內 360° ∨ 在前方 `coneHalfDeg`(45°) 視錐內）。**視錐距離無限、靠牆截斷**；senseRadius × 單位 strength。
- 玩家畫面：**離屏 canvas 遮罩**（`fogCanvas`/`fctx`）：填霧 → `destination-out` 挖視錐扇形(`visionPolygon` 射線扇)＋近身圈 → `ctx.drawImage` 貼回主畫布。可見孔露世界、其餘壓黑；霧上再疊淡牆骨架(`drawSkeleton`)當空間參考。`G` 除錯時關霧。
- AI 也吃同一套 `canSee`：敵人/待機只對看得到的目標開火、繞錐外＝看不到。
- ⚠️ 踩雷紀錄：**不可在主畫布直接 destination-out**（會擦掉世界露出底色，敵人/子彈消失但血條還在）——必須走離屏遮罩。

### 4.4 武器 / 開火（發散 bloom）/ 換彈 / 彈藥
- `fireUnit(u, angle, spreadDeg, fireRateMult, bloomAdd) → bool`（回傳是否真開槍，供 TAP 計發數）。
- **手動/敵人** `manualFire/enemyFire`：spread = `spreadBase + u.bloom`，bloomAdd = `spreadPerShot`（連射越散、停火 `spreadRecover` 回復）。玩家點放準、噴射散＝技巧。
- **待機** `idleEngageFire`：spread = `spreadBase/idleAccuracyMult + u.bloom`（基礎較差），射速 ×`idleFireRateMult`，會累積 bloom。
- 換彈：`magazine/reloadTime/reserveAmmo`，空彈匣自動換、`R` 手動換；`tickAmmo` 處理換彈完成補彈 + bloom 衰減。**備彈暫訂 9999**（基本不咬人）。
- **Shift 奔跑** `sprintMult(u) = clamp(base − (weapon.weight−1)×weightPenalty, minMult, base)`，只對被附身且移動中的單位；現值 SMG×1.90 / PDW×1.80 / Rifle×1.50。

### 4.5 待機個性矩陣（ARCHETYPES）
- 每名單位招募時隨機抽一個 archetype；`makePlayer` 由 archetype 衍生 `idleStrategy{fireMode,targeting:'NEAREST_IN_LOS',facing,discipline}`。
- **視線軸 facing**：`HOLD`(維持 idleAnchor) / `SWEEP`(以 idleAnchor 為中心 ±sweepAmpDeg 掃視) / `WANDER`(只在 idleAnchor ±`wanderAmpDeg`(28°) 內、以 `wanderSpeed`(45°/s) 緩慢小幅擺動) / `TRACK`(轉向看得到的最近威脅)。
- **idleAnchor**＝單位「進入閒置那刻的正前方」，在 CONTROLLED→IDLE（possess）與 toggleExtract 時設定；SWEEP/WANDER 以它為中心、HOLD 維持它、附身時恢復到它。
- **射擊紀律 discipline**：`AUTO`(扣到底：持續噴、bloom 拉滿、耗彈快、壓制) / `TAP`(點放：`idleTap{size:3,gap:0.55}` 短 burst + 停頓讓 bloom 回復、準/省彈/斷續)。在 `idleTick` 內實作 burst 計時(`burstLeft/burstCd`)。
- 8 種（tier 為盯哨可靠度，僅用於卡片名稱顏色 `TIER_COLOR`）：Rock(HOLD/TAP/AGG ★★★)、Sentinel(SWEEP/TAP/AGG ★★)、Suppressor(HOLD/AUTO/SUPP ★★)、Hunter(TRACK/TAP/AGG ★★)、Sprayer(SWEEP/AUTO/AGG ★)、Watchdog(TRACK/AUTO/DEF ★)、Drifter(WANDER/TAP/DEF ☆)、Twitch(WANDER/AUTO/AGG ☆)。
- 設計目的：把「盯哨可靠度」變成抽人的風險（飄忽的人逼你勤切勤盯；穩的人可放著）。與迷霧協同（WANDER/SWEEP 視錐會轉離威脅＝真的看不到）。
- `idleTick(u,dt,lockFacing)`：`lockFacing=true` 時保留開火但不改朝向（撤離用）。

### 4.6 敵人 AI（waypoint 圖，不尋路）
- `enemyTick`：每 frame 在 waypoint 圖選點。候選＝當前節點+鄰居，排除被別人 reserve 的特殊節點。
- `scoreNode` = tag 權重 + 朝最近我方推進項(×`chargeMul` 未交火 / ×`holdProxMul` 已交火) + 側翼角項(Flanker) + 暴露懲罰(被我方 `canSee` 的節點) + 視線項(查 `nodeVis`)。
- **hysteresis**（`hysteresisBonus`，目標節點加分防抖；`H` 鍵切換對照）。
- **reservation**（特殊節點 flank/vantage/choke：選擇當下就佔、改目標/死亡即釋放，不永久鎖死）。
- 開火/朝向受感知限制（`canSee`）；**堅守時視錐朝 `watchPoint`＝最近且朝我方那側的 waypoint**。
- 性格：Rusher / Flanker（`CONFIG.ai`）。

### 4.7 撤離與軟計時
- `extractTick`：沿 waypoint 圖 next-hop 自走到撤離點（`nextHop` 由撤離節點 BFS 預算）；**面向移動方向**（rotToward 朝 goal）+ `idleTick(u,dt,true)` 保留 overwatch 開火（不閃避不繞路）。
- 跟隨鏡頭：`followExtractId` 時 `camTarget` 跟位置且角度＝移動朝向（朝上）。觀看結束＝按任一非換人鍵 / 該單位撤出或陣亡。
- 軟計時：`t > timeBudget(90s)` 後從進入點持續 spawn Rusher 追兵（間隔遞減），直到場上無我方。
- down=dead 永久移除；場上無 active 單位即 `endMission()`。

### 4.8 路線圖
- `ROADMAP`：手刻 DAG（10 節點、5 直行、COMBAT/REST 兩型；起點 2 個、終點 1 個 COMBAT）。`selectNode`（滑鼠點 `roadNodeAt` 或數字鍵）→ 進節點；`advanceNode` 標記完成、開放後繼，無後繼＝RUNWIN。
- 難度：每清一關（`missionIndex`）多放 1 名 rusher（暫定）。

### 4.9 掉落/拾取（戰鬥內）
- `killUnit` → `spawnDrop`：我方/敵方陣亡單位的武器落在原地（底線武器不掉）；drop 帶 `ammo`（折算量：敵 `salvageEnemy`、我 `salvageAlly`）。
- `updateDrops`（每幀、step6）：**只有被附身單位**走到 `pickupRadius` 內自動撿進 `backpack`（任務中不可動用＝撿/不撿的時間/暴露取捨，無管理）。`drawDrops` 在世界層（受迷霧）。HUD 顯示 BACKPACK/地上件數。
- 掉落只存活於該任務；撿進背包者跨任務累積，直到下個 REST 統一整理。

### 4.10 休息關 meta 流程（共用倉庫 + 二選一 + 配裝 + 彈藥分配）
- 進入：`enterRest` 整理背包（彈藥進儲備、item 進倉庫 haul，可能超上限）。`restPhase`：`CHOICE`(可整理) → `OFFERS`(搜刮三選一)/`TARGET`(指定升級對象) → `DONE`(可整理→繼續)。
- **搜刮**（`restDoScavenge`→`rollScavenge`）：`scavengeChoices`(3) 抽選，**地板保證 offers[0]＝supply**（基本彈藥補給，絕不空手）；天花板可出 weapon/ability(→倉庫)、maxhp(全隊)、personality/discipline(需 `TARGET` 指定成員)。`pickOffer`→`applyOffer`（needTarget 者轉 TARGET→`applyTarget`）。先抽取得→再整理倉庫（acquire-then-sort）。
- **喚醒**（`restDoAwaken`）：**`overCap()` 時禁用**（§5 時序：先把倉庫整理到 ≤ 當前上限才能喚醒，防止用 +slot 回溯救回該丟的東西）。+1 隊員（cap +slotsPerMember）→ `autoEquipFromWarehouse`（取估值更高的倉庫槍，否則保留 makeRecruit 的手邊槍；皆無則底線）。
- **繼續**（`restDoContinue`）：`DONE` 且 `!overCap()` 才放行 → `advanceNode`。
- 配裝（CHOICE/DONE 皆可）：`equipWeaponFromWarehouse`/`unequipWeapon`(→底線 pistol)/`equipAbilityFromWarehouse`/`unequipAbility`/`discardWarehouse`/`allocAmmo`。底線武器換下即消失（不回倉庫）。
- UI＝`drawRest` 重建 `restRects`（draw 階段註冊可點區域）；`restClick` 反向命中最上層。`drawLoadoutCard` 每員顯示**個性(§8 配槍前看覆蓋力)+武器+技能+彈藥±**；`drawRestWarehouse` 倉格(超上限紅框)+丟棄；`drawRestActionBar` 二選一/繼續；`drawRestOffers` 模態三選一。鍵盤＝便捷（CHOICE 1/2、OFFERS/TARGET 1‑9、DONE Enter）。

### 4.11 多張戰鬥地圖 + 地形（門 / 低矮牆 / 按鈕）
- **地圖 registry**：`MAPS.alpha`(原圖)/`MAPS.bravo`(門/掩體圖)；`let MAP` 為作用中地圖。衍生表(`N/adj/nodeVis/nextHop/EXTRACT/ENTRY/extractNodeId`)改由 `buildMapDerived(key)` 重建（原本是載入時 IIFE）。`startMission` 依 `roadNode(roadCurrent).map||'alpha'` 切換並 `for(door) open=false` 重置。ROADMAP COMBAT 節點加 `map` 欄。
- **門**：`MAP.doors=[{id,x1,y1,x2,y2,open}]`。關＝實體（擋移動/視線/子彈）；開＝無。視線/子彈經 `segBlocked`(牆+關門) 截斷；`nodeVis` 用 `clearLineWalls`(只看永久牆，與門開關無關、穩定)。
- **低矮牆**：`MAP.lowWalls=[{x1,y1,x2,y2}]`。**不擋視線**；玩家穿越減速(`lowWallSpeedMult`，控制移動與 `extractTick` 都套)；敵人視為實體（`collideWalls` 對 enemy 推開→靠 waypoint 邊繞行，**邊不得穿越低矮牆**＝地圖設計約束，已用無頭測試把關）；子彈＝`b.ox/oy` 發射原點到低矮牆 `<closeRange` 可越射、否則 `blockChance` 機率擋。
- **按鈕**：`MAP.buttons=[{id,x,y,doorId}]`，遠端連動門（showcase：bravo b1 開 door2 通往撤離）。
- **互動**：`tryInteract`（E 鍵；僅被附身單位）找 `interactRadius` 內最近門/按鈕→toggle。`nearestInteractable` 也供 `drawScreenOverlays` 顯示「E：開/關門/啟動」提示。**自走單位(撤離/閒置)不會開門**（§Q4）→ ⚠ 關門卡撤離路有 softlock 風險，故 bravo 設計讓門在側/中段、玩家推進時自然開好。
- 繪製：`drawDoors`(關=粗琥珀/開=淡虛線)、`drawLowWalls`(藍灰虛線)、`drawButtons`(連動門開=綠/關=琥珀)，並加進霧上 `drawSkeleton` 空間參考。

### 4.12 敵人狀態機（IDLE ↔ ENGAGED，latch）
- `enemy.state`：`IDLE`(守點被動：`enemyIdleTick` 只回正 `watchFacing`、不推進/不開火/不開門) ↔ `ENGAGED`(原選點/charge-hold/開火 + `enemyTryOpenDoors` 破門)。
- 進入 ENGAGED（**永不取消**）：① `enemyTick` 偵測 `canSee` 任一玩家 ② 玩家開火（`fireUnit` 內 `alertEnemiesNear`）`enemyHearRadius` 內、不論命中 ③ 被命中（`damageUnit` enemy 分支）。
- `makeEnemy` 預設 `state:'IDLE'`、`facing/watchFacing` 可由 `enemySpawn.facing` 指定（bravo 門後敵人面向 door＝預設 Math.PI 朝左）。**追兵(reinforce) 直接 ENGAGED**。
- bravo 玩法：開場敵人靜止守點、藏在關門後（門擋視線→霧中看不到），開門才現形/交火。

### 4.13 關卡編輯器（scene EDITOR）
- **進入/離開**：路線圖右上「✎ 關卡編輯器」鈕（`editorBtnRect`，MAP click 命中）→ `editorOpen`；編輯器內「離開」鈕或 Esc → `editorExit`(回 MAP)。`edit` = 編輯中地圖（同 MAPS 結構，**waypoints id===index**，刪點會 `removeWaypoint` 重編號並修正 edges/spawns）。
- **工具**(數字 1‑0 或工具列)：wall/door/low(拖拉線段 `editorCommitSeg`)、wp(點放)、edge(點兩 waypoint 連線 `toggleEdge`)、spawn(點 waypoint 循環 rusher→flanker→無 `cycleSpawn`)、button(放置並自動連最近門)、entry/extract(設點)、erase(就近刪/右鍵亦可)。`editTagBrush` 決定新 waypoint 的 tag。`editSnap`(G 鍵)格線吸附。
- **輸入**：`editorMouseDown`(先命中 `editRects` UI，否則畫布放置/起拖) / `editorMouseUp`(落筆拖拉線段)；座標 `snapPt`。
- **即時驗證** `editorValidate`：缺 waypoint/進撤點、edge 穿低矮牆數、進場↔撤離未連通——底部狀態列顯示，且擋掉不可玩的試玩。
- **試玩** `editorTestPlay`：把 `edit` 正規化存成 `MAPS.__editTest`，`forcedMap='__editTest'`+`fromEditor=true`，`startMission()` 直接打（不動到 run 的 roster/missionIndex/roadmap）；結束(`endMission` 首行)或 Backspace 中止 → 回 EDITOR。
- **輸出** `editorExport`：把 `edit` 序列化成可貼回 `MAPS{}` 的 JS 字面量 → `console.log` + `localStorage('sleeper_map')`；之後在 ROADMAP 某 COMBAT 節點加 `map:'鍵名'`。`editorLoad('blank'|'alpha'|'bravo'|'saved')` 可載入起點。
- 繪製 `drawEditor`/`drawEditGeom`/`drawEditorUI`/`eBtn`：渲染 edit 幾何(edge 穿矮牆顯紅)+工具列/標籤/動作鈕+拖拉預覽+吸附游標+驗證列。
- **Phase 3 擴充**：工具新增 `prisoner`(P 鍵/工具列「俘虜」)放置俘虜點；工具列加「時間:NNs」循環(0預設/60/90/120/150)設 `edit.timeBudget`；`editorExport` 帶出 `prisoners` 與（非 0 時）`timeBudget`。

### 4.14 裝備 / 道具（每單位各 1 槽，可未來擴充）
- `EQUIPMENT`（持續被動，`fx` 效果於 `makePlayer` 套用並存成 per-unit 修正）：maxHpAdd / dmgResist(減傷) / senseMul / reloadMul / magMul(彈夾) / spreadMul(發散) / turnMul / moveMul。現有 armor/scope/quickmag/extmag/stabil/light/plate。

- roster entry 加 `equipId`/`itemId`；`makePlayer` 衍生 `dmgResist/reloadMul/spreadMul/turnSpeed/moveMul/magazine/buffT/buffMove/buffFire`；消費端：`damageUnit`(減傷)、`startReload/tickAmmo`(`unitMag`/reloadMul)、`fireUnit`(spreadMul/buffFire)、移動(moveMul/buffMove)、turn(turnSpeed)。

- **裝備加的 maxHp 不計入 `bonusMaxHp`**（`rosterFromPlayer` 扣掉 `fx.maxHpAdd`），避免卸裝後血上限殘留。

### 4.15 軍械庫節點 / 俘虜 / 各關時間
- **軍械庫**（ROADMAP type `ARMORY`，node 2/8）：`enterArmory`→`rollArmory` 三選一(武器/裝備/技能/道具)→`armoryPick` 入倉庫→`advanceNode`。獨立 scene `ARMORY`（`drawArmory`，重用 `restRects`/`restClick`；鍵盤 1‑9）。不進戰鬥、不耗時。

## 8. 設計決策與取捨（脈絡：為什麼這樣）

1. **待機可信但不足**：待機開火基礎較差（spreadBase/0.5）、慢射速、反應延遲、只看 90° 錐+近身；個性決定可靠度。放著不管終會被繞側折掉、單靠待機打不贏整圖。
2. **chargeMul/holdProxMul（超出原規格的敵 AI 旋鈕）**：依「敵人能否 `canSee`+射程交火」切換推進權重——不能交火就推進、能交火就拉開距離對射，避免在射程外卡點或自殺貼臉。**調敵人侵略性從這兩個下手。**
3. **bloom 手動專屬、待機另計**：早期把 bloom 套到待機導致待機噴 30° 在 12 秒被秒；現分流（手動 bloom＝技巧、待機固定基礎+bloom 但 TAP 靠停頓壓住）。
4. **恆朝上 + FPS mouselook**：見 §4.2 取捨。
5. **迷霧離屏遮罩**：見 §4.3 踩雷。
6. **全隨機招募、無基準**：使用者接受失去固定基準，換取每輪 run 不同；個性＝抽人的風險層。

## 12. 驗證方式與踩雷

- **無頭測試法**（每次改完都跑）：抽出 `<script>`，在 Node `vm` 沙箱 stub 掉 `document/window/performance/requestAnimationFrame` 與 Proxy canvas ctx（離屏 canvas 用 `document.createElement` stub、需 `document.exitPointerLock` stub、fakeCanvas 需 `style:{}`），移除結尾 `requestAnimationFrame(frame)` 後逐幀驅動。
- **務必模擬真實迴圈**：在所有場景（含 MAP）逐幀跑 `update()+draw()`，否則會漏掉「非戰鬥場景碰 cam/players」這類崩潰。
- 踩雷清單：①`update` 必須先判 `scene!=='COMBAT'` 再碰 `cam/players`（MAP 時 cam 未建立）。②迷霧不可在主畫布 destination-out（要離屏）。③TTK 旋鈕請見「§8 設計取捨」與下方。
- 典型基準：放置小隊與第一波對拼後殘存者撐到追兵湧入才被折（可信但不足）；hysteresis ON 明顯降低敵人選點抖動。

### TTK 旋鈕速查（之前討論過）
- 單體爆發 TTK：目標 HP、`weapon.damage × fireRate`。
- 有效 TTK：發散(spread/bloom)、range、projSpeed。
- 持續 TTK：magazine/reloadTime（備彈 9999 故目前只有換彈空檔咬人）。
- 交火 TTK：反應延遲、迷霧/朝向、手動 vs 待機(idle 三倍率)、集火數。
- 注意搜刮會讓我方 maxHp 一路 +15 累積，run 後段我方 TTK 變長。
