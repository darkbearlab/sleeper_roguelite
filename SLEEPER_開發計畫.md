# Codename: SLEEPER — 開發計畫

> 工作標題，之後再換。一句話定位：**俯視戰術射擊 × 附身式小隊操作 × Slay the Spire 式 roguelite。你一次只能完整附身一具身體，其餘隊員靠固定本能運作；每張圖在時間關門前，至少把一個人帶出撤離點。**

本文件是給 Claude Code 的建置規格。所有數值皆為**佔位值**，集中在 §12，先求跑得動、手感對，再調平衡。技術目標沿用既有慣例：**單一 HTML 檔、原生 JS、Canvas 2D，無外部函式庫、無外部資源**。

---

## 1. 設計憲章（不可違反的核心原則）

這些是整個設計的承重牆，實作時優先級高於一切便利性。任何「優化」若違反以下任一條，視為錯誤：

1. **待機 AI 必須「可信但不足」。** 待機能讓單位活著（壓制、看守視線），但**贏不了仗**。待機開火比手動弱（精度低、反應慢、不集火優先目標、不放技能、不閃避）。玩家提供精準與走位，AI 只提供分擔注意力。**嚴禁把待機開火做得跟手動一樣好。**
2. **待機策略是每個角色固定、玩家不可更換的。** 它是角色的「人格」，也是 build 的一部分。可換的只有武器/裝備（在休息關處理）。
3. **切換角色沒有冷卻。** 摩擦來自「同時只能瞄一個人」這個注意力上限本身，不是人工 CD。（唯一例外：可加一個極小的交接延遲僅為防止輸入抖動，預設 0。）
4. **時間是軟計時，不是失敗畫面。** 時間到 → 追兵從進入點湧入，不是 Game Over。計時器的角色是**貪婪調節器**。
5. **倒下即死，永久損失，無傷員/復活。** 壓力前移到「預防」。核心技術 = 看出哪個待機單位快被折了、在它死前切過去。
6. **勝負條件極簡：** 一張圖**至少 1 人撤出 = 過關，推進下一關**；**0 人撤出（全死在場上，含被追兵清掉）= 該 run 結束。** 每個撤出的單位帶到下一關（多一把槍）。
7. **難度是 emergent 的**：敵人佈置 + 進入/撤離點的地理 + 玩家選擇多貪 + 帶進來的陣容大小。不要靠單一數值線性堆難度。
8. **地圖與敵人佈置是手刻的。** 敵人靠手刻的 waypoint 圖移動，**不寫空間尋路（A\*/navmesh）**。

## 2. 核心迴圈

**逐秒（任務內）：**
附身一名單位 → WASD 移動 + 滑鼠瞄準/開火 + 技能。其餘單位跑固定待機策略（看守、視線內自動開火、通常不動只轉向）。用熱鍵切換附身對象；切走的單位立刻回到待機。實質玩法 = **在隊員間分時切換，把火力佈署到地圖上，同時盯住快撐不住的那個。**

**單張圖：**
潛入 → 清出/壓制通往撤離點的走廊 → 開始讓單位撤離（逐人/逐組）→ 時間關門前盡量多撈人出去。可選：花時間繞道撈資源/俘虜。

**run（路線圖）：**
Slay the Spire 式節點路線圖。節點類型：戰鬥、休息關、（部分）含可選經濟的戰鬥。玩家選路。陣容、技能、武器隨機獲取，**變異來自 build 而非地圖**（地圖靜態）。

**單一資源貫穿全局：時間/注意力。** 它在三個尺度反覆出現同一張力——逐秒（顧哪個單位）、單圖（多�leave幾個人/撈不撈東西）、路線圖（走安全還是貪婪）。

## 3. 操作與附身模型

- 移動：`WASD`（相對螢幕，純俯視）。
- 瞄準/開火：滑鼠移動瞄準、左鍵開火。
- 技能：`右鍵` / `Space` / `Shift`（佔位，依角色技能槽）。
- 切換附身：數字鍵 `1..N` 直接選人；`Tab` / `Q`/`E` 循環（擇一實作，預設數字鍵）。
- 設定撤離：對選定單位按 `F`（切換 extract on/off）。
- 系統狀態：每個單位有 `controlState ∈ { CONTROLLED, IDLE, EXTRACTING, EXTRACTED, DEAD }`。
  - 同時只有一個單位是 `CONTROLLED`。
  - 切換附身：舊的 → `IDLE`，新的 → `CONTROLLED`。
  - `EXTRACTING` 的單位自動沿 waypoint 圖往撤離點走（見 §8），可被「抓回」（再次按 `F` 或附身它）變回 `IDLE/CONTROLLED`。

## 4. 待機策略系統

待機策略是一份固定 config（每角色綁定，玩家不可改）。Tick 行為：

```
IdleStrategy {
  fireMode:   AGGRESSIVE | DEFENSIVE | SUPPRESSIVE | HOLD_FIRE
  // AGGRESSIVE  = 主動射視線內任何敵人
  // DEFENSIVE   = 僅在自己被瞄準/受擊後反擊
  // SUPPRESSIVE = 朝指定方向錐持續壓制（命中與否其次）
  targeting:  NEAREST_IN_LOS | MOST_EXPOSED | FIRST_SEEN | NONE
  facing:     HOLD | SWEEP(angle,speed) | FACE_NEAREST_THREAT
  moveOnIdle: false   // 預設不動；僅特定 buff 可解鎖（如「受擊後退一步進掩體」）
}
```

待機開火一律套用**待機懲罰**（見 §12：精度倍率、反應延遲、射速倍率），且**不放技能、不閃避、不主動換位**。待機只能讓你不死，不能替你打贏。

不同角色 = 不同 IdleStrategy 組合，形成各自的「自動駕駛人格」。換一套陣容 = 換一套自動覆蓋，同一張圖會玩出不同解。

## 5. 角色 / 武器 / 技能 資料模型

內容皆佔位，先定資料形狀。建議全部以 JS 物件/JSON 內嵌於 HTML。

```
Character {
  id, name,
  idleStrategy: IdleStrategy,      // 固定不可換
  stats: { hp, moveSpeed, turnSpeed, hitRadius },
  weaponId,                        // 可在休息關更換
  abilityIds: [],                  // 可在休息關更換
}

Weapon {
  id, name,
  damage, fireRate, range, spread, projectileSpeed,
  magazine, reloadTime            // 彈藥系統可先簡化或關閉
}

Ability {
  id, name, cooldown, effect       // 佔位：衝刺、煙幕、集火標記… 僅手動可用
}
```

**Buff（roguelite 強化）分兩軸**，資料上明確標記它影響哪一軸：
- `AUTOPILOT` 軸：強化待機行為（待機也會預判提前量／血量低於門檻自動退入掩體／待機可放某被動技能…）。
- `HANDS` 軸：強化手動操作（手動瞄準時增傷／解鎖衝刺／換彈加速…）。

兩軸對應「帶大隊吃自動」與「帶小隊吃手感」兩種流派。

## 6. 敵人 AI（最關鍵、最易做錯的部分）

敵人不做空間尋路。智能 = **在手刻 waypoint 圖上選下一個節點**。

### 6.1 Waypoint 圖
```
Waypoint  { id, pos, tags: [] }     // tags 例：cover, flank, vantage, rush, choke, patrol, spawn
WaypointEdge { a, b }               // 保證 a,b 之間有直線可走且有視線；敵人 node→node 直線移動即可
```

### 6.2 性格 = 選點策略
```
Personality {
  preferredTags: { cover: w, flank: w, vantage: w, ... },  // 權重
  scoringTerms: [...],   // 依玩家/我方位置打分（見下）
  hysteresisBonus,       // 當前目標的黏著加成（防抖，必填）
}
```
起手包性格（佔位內容）：
- **Rusher**：偏好 `rush`，評分 = 朝最近我方單位推進。
- **Flanker**：偏好 `flank`，評分 = 繞到我方側/後、且**避開暴露在我方火力錐的節點**。
- **Holder**：偏好 `choke`/`cover`，評分 = 卡住我方前進軸。
- **Marksman**：偏好 `vantage`，評分 = 找到能看到我方且自身暴露低的點，到位後不動持續射擊。

### 6.3 評分與更新（每 frame）
每個敵人每 frame：
1. 蒐集候選節點，**排除已被其他單位 reserve 的特殊節點**。
2. 對每個候選節點打分：`tag 權重 + 各 scoringTerm（依玩家位置）+ 火力暴露項`。
3. **對「當前目標節點」加上 `hysteresisBonus`**（黏著），新節點要明顯更高才換。→ **這一條是防止逐 frame 重評造成原地抖動的關鍵，務必實作。**
4. 取最高分節點為目標；若換了目標：釋放舊 reservation、（若新節點為特殊節點）reserve 新節點。
5. 朝目標節點直線移動；對視線內我方單位依武器開火。

### 6.4 Reservation（特殊節點佔用）
- 在**選擇當下**就 reserve（不是抵達才算），避免兩隻挑同點半路相撞。
- 其他敵人評分時排除已被 reserve 的特殊節點。
- **釋放規則（必做，否則節點被永久鎖死）**：單位改變目標、死亡、或離開該節點時，立即清掉其 reservation。
- 只有特殊節點（vantage/flank/choke）需要 reserve；一般 `patrol`/路徑點放任多隻共用。

### 6.5 視線/可見度
- 地圖靜態 → **載入時預算一張「節點對節點可見度表」**（對所有需要的節點配對 raycast 牆面一次）。敵人的「避開暴露在我方火力的節點」評分、以及我方待機 overwatch 的視線判定，盡量查這張表，避免每 frame raycast。
- 單位↔單位的即時視線（少量單位）用 runtime raycast 即可。

## 7. 地圖資料格式

```
Map {
  walls: [ {x1,y1,x2,y2}, ... ],          // 線段或多邊形邊
  waypoints: [ Waypoint ],
  edges: [ WaypointEdge ],
  entryPoints: [ {pos} ],                  // 玩家進入點，也是追兵 spawn 點
  extractionPoints: [ {pos} ],
  enemySpawns: [ {pos|waypointId, enemyType, personality} ],
  objectives: [ Prisoner|Resource ... ],   // 可選經濟，見 §9
  timeBudget,                              // 佔位
}
```
- 進入點離撤離點的距離 = 該圖「延長賽」有多兇的天然旋鈕（追兵從背後追，距離越短越殘）。
- 預算可見度表可在載入時由 walls + waypoints 算出，不必存進地圖資料。

## 8. 撤離與軟計時

### 8.1 撤離（per-unit）
- 任一單位**以任何方式到達撤離點**（手動走上去，或被設為 `EXTRACTING` 後自動沿 waypoint 圖走過去）即 `EXTRACTED`，從場上移除並**入帳到下一關陣容**。
- `EXTRACTING` 自動導航：走笨版（沿圖直線移動 + 保留待機開火，不閃避不繞路），可被抓回。
- 撤離路由可選：**快路**（最短、暴露）或**穩路**（偏好 `cover` 標籤、較慢）。先實作快路，穩路可後補。
- 玩家可一邊附身某單位當殿後/掩護，一邊讓其他人 `EXTRACTING`。

### 8.2 軟計時 → 追兵
- `t > timeBudget` 後，從**進入點**開始持續 spawn 追兵（Rusher 性格，撲最近我方），**持續加碼、永不停止，直到場上無我方單位**。→ 保證「留著交戰必死」。
- 追兵從背後來，效果是把玩家往撤離點趕，並優先咬到留在後方的單位。

### 8.3 關卡結束與勝負
- 當**場上不再有我方單位**（全部 `EXTRACTED` 或 `DEAD`）→ 結算。
- `EXTRACTED ≥ 1` → 過關，帶著入帳陣容進下一關。
- `EXTRACTED = 0` → run 結束。
- 「衝一個人直奔撤離點、其他不要」是合法的斷尾逃生（代價：下一關槍少），不是 bug。

## 9. Roguelite 結構與補員經濟

### 9.1 路線圖
Slay the Spire 式節點地圖，玩家選路。節點：戰鬥 / 休息關 / 含可選經濟的戰鬥。

### 9.2 休息關（安全節點）
- 二選一：**搜刮物資**（強化現有單位/裝備，go-tall/微操向）或 **喚醒 sleeper**（+1 單位入隊，go-wide/自動向）。
- **裝備調整常駐可用**（重新分配手上的武器/技能）。
- 喚醒 vs 搜刮 = 直接對應「人數＝自動/手動光譜」，休息關是玩家校準整個 run 走哪一端的地方。
- **休息關刻意稀少**，使任務內經濟不致多餘。

### 9.3 任務內經濟（折進時間貪婪）
- 部分戰鬥圖放置可選的資源/俘虜，**撈取花時間**（時間 = 同一個貪婪貨幣）。
- 擺在**偏離主撤離線的繞道**上 → 撈它同時花時間 + 吃一段暴露繞路。
- **俘虜 = 護送酬載**：必須由玩家護送、且**指名這一個**走到撤離點才入帳（與我方「任一個出去就過關」不同）。俘虜可不會打、純拖油瓶，把撤離物流難度往上推一階。

### 9.4 經濟平衡
- 來源（休息關 + 任務內撈）對上 支出（down=dead 損耗）。這條 source/sink 比例是主要平衡目標，先用佔位值，確保陣容不會單調遞減成死亡螺旋。

## 10. 視覺與呈現

- 純俯視，風格參考 Door Kickers：乾淨、機能性、可讀性優先。Canvas 2D。
- 必要可讀元素：單位朝向與**視線/火力錐**（待機單位的覆蓋一目了然）、被附身單位的高亮、撤離點、進入點、追兵來向、計時器、各單位狀態（含「快死了」的明顯警示——這是核心技術所在，視覺上必須讓玩家來得及反應）。
- 美術先用幾何圖形佔位。

## 11. 技術架構

- **單一 HTML 檔，原生 JS，Canvas 2D**，無外部相依。
- 固定時間步主迴圈（update / render 分離）；建議 `requestAnimationFrame` + 累積器固定 dt。
- 系統劃分（模組化但同檔）：`Input`、`Possession`、`IdleAI`、`EnemyAI`（含 reservation 與評分）、`Combat/Projectiles`、`LoS/Visibility`（含預算表）、`Extraction`、`Timer/Reinforcements`、`MapLoader`、`Render`、`MetaState`（路線圖/陣容/buff，phase 2 才接）。
- 地圖、角色、武器、敵人資料以內嵌物件存放，方便手刻與調整。

## 13. 建置順序（里程碑）

**先做一個能玩的垂直切片，證明核心手感，再疊 meta。**

- **Phase 1 — 垂直切片（單圖核心迴圈）**
  1. Canvas 主迴圈 + 純俯視渲染（幾何佔位）。
  2. 一張手刻地圖（牆、waypoint 圖含 tags、進入點、撤離點、幾個敵人 spawn）+ 載入時預算可見度表。
  3. 3 個 player 單位、各自 IdleStrategy；附身 + 熱鍵切換；手動移動/瞄準/開火（先不放技能或放 1 個）。
  4. 待機 FSM（套用待機懲罰的 overwatch 開火）。
  5. 敵人 AI：2 種性格（Rusher + Flanker）、每 frame 評分 + **hysteresis** + **reservation（含釋放）**。
  6. down=dead；撤離（走上撤離點/`F` 自動導航）+ per-unit 入帳 + ≥1 過關判定。
  7. 軟計時 → 從進入點湧出追兵。
  → **驗收：能在一張圖把人帶出去、能感受到「分時切換 + 待機接管 + 趕在追兵前撤離」的張力。**

- **Phase 2 — roguelite meta**
  路線圖、休息關（搜刮/喚醒/調裝）、陣容跨關攜帶、buff 兩軸、任務內資源/俘虜、多張手刻地圖、source/sink 經濟。

- **Phase 3 — 內容與打磨**
  更多敵人性格、武器、技能、穩路撤離、可見度與 AI 調校、平衡、視覺可讀性強化（尤其「快死了」警示）。

## 14. 驗收標準（節錄）

- **待機可信但不足**：待機單位能靠 overwatch 撐住短時間，但放著不管終究會被繞側折掉；單靠待機打不贏一張圖。
- **切換零摩擦**：連續快速切人不卡頓、無 CD 懲罰。
- **敵人 AI 不抖**：逐 frame 重評但不原地鬼步（hysteresis 生效）；不會兩隻擠同一個特殊節點（reservation 生效）；reservation 不會永久鎖死。
- **軟計時非失敗畫面**：時間到只是追兵湧入；只要 1 人已出去仍算過關。
- **貪婪張力成立**：玩家會在「再撈一個 vs 見好就收」之間真的猶豫。
- **down=dead**：倒下即永久損失，核心壓力在「在它死前切過去」。
