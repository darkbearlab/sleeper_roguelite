# 新地圖編輯器 — 規劃（待核可後實作）

> 狀態：**提案**。依使用者指示「地圖編輯器先規劃、核可後再更新」。本文是核可後的實作依據。
> 對照：現有的「遊戲內編輯器」(scene EDITOR，見 開發進度與脈絡.md §4.13) 為畫布手繪、單檔內，較擠且連線編輯薄弱；本提案改為**獨立檔 DOM 工具**。

## 1. 目標與範圍（來自需求）
- 獨立工具，**可匯出與讀取（再次編輯）**編輯結果。
- **三分頁**：① 地圖物件 ② waypoint 與連線 ③ waypoint 上的敵人配置（**不改遊戲現有生成方式**＝沿用 `enemySpawns`）。
- 地圖物件與 waypoint 皆可**拖曳調整**；**連線編輯要特別下功夫**。
- 匯出的地圖可再匯入編輯。
- 含**試玩**功能（因此放不同檔較合理）。

## 2. 架構決策（建議）
- **新檔 `editor.html`（獨立、自足）**：自帶畫布＋DOM 介面＋編輯邏輯，可單獨開啟。
- **`index.html` 維持「單一檔、開檔即玩」**——遊戲本體不依賴編輯器。只加一個**極小的試玩接縫**（`?test=1` 時從 localStorage 讀地圖、直接進一場戰鬥；正常開檔路徑完全不變）。
  > 「單一檔」原則是針對**可出貨的遊戲本體**；編輯器是開發工具，獨立檔不違反此原則。
- **資料回流遊戲**：編輯器「匯出 JSON」→ 你貼回 `index.html` 的 `MAPS`（保持 index 自足）。另提供「複製為 MAPS 片段(JS)」一鍵生成 `key: {…}` 方便貼上。
- **不動既有 in-index 編輯器**：兩者先並存；新編輯器成熟後再考慮淘汰舊的。

## 3. 資料格式（與遊戲 `MAPS` 完全一致 → 可雙向 round-trip）
編輯器的 in-memory 模型 = 遊戲地圖物件本身，匯出即此 JSON：
```
{
  label: string,
  timeBudget?: number,                       // 省略＝用全域 CONFIG.timeBudget
  walls:        [{x1,y1,x2,y2}],
  doors:        [{id, x1,y1,x2,y2, open?}],  // id 供按鈕連動
  lowWalls:     [{x1,y1,x2,y2}],
  bushes:       [{x,y,w,h}],
  destructibles:[{x1,y1,x2,y2, hp, intact, broken}],   // intact/broken ∈ DESTRUCT 形態名
  buttons:      [{id, x, y, doorId}],        // 遠端開門
  waypoints:    [{id, pos:{x,y}, tags:[]}],
  edges:        [[a,b], …],                  // 無向（遊戲雙向加鄰接）
  entryPoints:      [{x,y}],
  extractionPoints: [{x,y}],
  enemySpawns:  [{nodeId, type?, personality?, facing?}],  // type∈ENEMY_TYPE_IDS;省略type＝grunt
  prisoners:    [{x,y}],
}
```
- **匯入**＝貼上/讀檔/選範本，解析成同結構。匯出/匯入互為逆。
- 編輯器需要的少量目錄（`ENEMY_TYPE_IDS`、waypoint 標籤清單、`DESTRUCT_PRESETS`、`DESTRUCT` 形態名）在 editor.html 內各放一份小常數（與 index 同步維護；數量少、變動低）。

## 4. 共用畫布 + 三分頁模型
- **單一世界畫布**所有分頁共用，支援 **平移(空白鍵拖/中鍵) + 縮放(滾輪)**（預留「地圖可更大」）。座標皆世界座標。
- 切分頁＝切換「**哪一層可互動/可編輯**」與右側屬性面板；其餘層淡色顯示作為對位參考。
- 共用：格線吸附(可調格距/開關)、復原/重做(undo/redo stack)、即時驗證列。

## 5. Tab 1 — 地圖物件
- 工具：選取 / 牆 / 門 / 低矮牆 / 破壞物(+預設下拉) / 草叢 / 按鈕 / 進場點 / 撤離點 / 刪除。
- **繪製**：線段類(牆/門/矮牆/破壞物)拖曳兩端點成線；矩形類(草叢)拖曳成框；點類(按鈕/進/撤)單點放置。
- **拖曳編輯**（選取工具）：拖整個物件移動；拖線段端點 / 矩形角點微調；點物件直接拖。
- **屬性面板**：選中物件的精確座標(可改數字)、破壞物預設/HP、門 `open` 預設、**按鈕→門 連動(doorId 下拉)**。
- 刪除：右鍵點物件 / 刪除工具 / 選中後 Del。

## 6. Tab 2 — Waypoint 與連線（**重點**）
- **waypoint**：放置自動編號(id)、拖曳移動(id 不變)、刪除時自動移除其相關 edges 並（選項）重編號。
- **連線編輯（多下功夫）**：
  - **拉線建邊**：從一個 node 拖到另一個 node 放開＝建立 edge；放開時有 rubber-band 預覽線跟著游標。
  - **點兩下建/刪**：點 node A 再點 node B＝切換該邊有無。
  - **刪邊**：滑鼠移到邊上高亮 → 點擊/右鍵刪除。
  - **節點檢視**：選中 node 顯示 id/座標/度數(邊數)/鄰居清單（點鄰居可跳選或一鍵斷開）。
  - **即時卡牆檢查**：邊若穿越牆 / 擋移動的破壞物 → 該邊畫紅 + 列入警告（沿用現有幾何規則，避免 AI 卡牆）。
  - 便利（選配）：依鄰近自動連線、孤點偵測高亮。
- **標籤**：waypoint 標籤(patrol/cover/flank/vantage/rush/choke/spawn) 以屬性面板勾選集設定（亦可標籤刷）。

## 7. Tab 3 — 單位配置（敵人 / 俘虜）
- **不改生成方式**：敵人＝掛在 waypoint 上的 `enemySpawns`。
- **指派**：點 waypoint → 設/循環 敵人類型(grunt/dog/heavy_lmg/heavy_shot/無)；設個性覆寫(rusher/flanker/holder…)、**朝向 facing**（拖一個方向把手或填角度）。
- **預覽**：在該 waypoint 畫敵人圖示＋朝向箭頭＋**視野錐預覽**（直接驗證哨兵守備角度/側翼缺口，呼應潛行/側翼設計）。
- **俘虜**：本頁放置/拖曳 `prisoners` 點（救援目標）。
- 屬性面板：每個 spawn 的 type/personality/facing 下拉與角度。

## 8. 驗證（即時、阻擋試玩/匯出時警告）
沿用並擴充現有 `editorValidate`：① ≥1 進場點、≥1 撤離點 ② 有 waypoint 且無孤立節點 ③ **edge 不穿牆/擋移動破壞物** ④ `enemySpawns.nodeId`/`buttons.doorId` 指向存在的目標 ⑤ 進/撤點不在牆內。畫面常駐檢查列 + 紅色標記。

## 9. 存讀 / 匯出匯入 / 試玩
- **本機草稿**：多個具名地圖存 `localStorage['sleeper_editor_maps'] = {name: mapJSON}`；新建/儲存/另存/載入/刪除/複製。
- **檔案**：下載 `.json`(Blob) / 上傳 `.json`(file input) — 讓地圖可離開瀏覽器、利於真正製作關卡。
- **匯出 MAPS 片段**：一鍵產生可貼回 index.html 的 `key: {…}` JS。
- **試玩**：序列化現圖 → 寫 `localStorage['sleeper_editor_testmap']` → 開 `index.html?test=1`(新分頁)。
  - **index.html 試玩接縫（唯一改動）**：載入時若 `?test=1` 且有該 localStorage → 註冊成臨時 `MAPS.__test`、`buildMapDerived('__test')`、用預設 roster 直接進一場 COMBAT（跳過序章/路線圖）；提供離開回到提示。正常開檔不受影響。

## 10. 與現有 in-index 編輯器的關係
- 先**並存**：舊編輯器(scene EDITOR)保留，新編輯器獨立。新版穩定後可把遊戲內入口導向新版或移除舊版（屆時再議）。

## 11. 實作里程碑（核可後）
1. `editor.html` 骨架：自足畫布 + pan/zoom/grid + 三分頁切換 + 共用模型/undo。
2. Tab 1 地圖物件（含拖曳端點 + 屬性面板）。
3. Tab 2 waypoint + 連線（拉線/點選/刪邊/節點檢視/卡牆檢查）。
4. Tab 3 敵人/俘虜配置（朝向把手 + 視野錐預覽）。
5. 驗證列 + 本機草稿 + 檔案匯出匯入 + 「複製為 MAPS 片段」。
6. index.html 試玩接縫 + 編輯器「試玩」鈕。
7. 驗證：editor.html 也用 headless harness 跑（DOM stub）；確認匯出 JSON 能被 index 的 `buildMapDerived` 正常吃下、幾何不卡牆。
